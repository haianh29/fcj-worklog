**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Exploring AWS services, brainstorming ideas, and selecting a project](1.1-Week1/)

**Week 2:** [Continuing AWS service exploration with hands-on practice](1.2-Week2/)

**Week 3:** [Starting the main project flow with temporary admin UI](1.3-Week3/)

**Week 4:** [Building temporary UI for the manager role](1.4-Week4/)

**Week 5:** [Starting API integration and aligning UI with backend contracts](1.5-Week5/)

**Week 6:** [Continuing API integration and refining UI to match backend](1.6-Week6/)

**Week 7:** [Starting project deployment using AWS services](1.7-Week7/)

**Week 8:** [Endpoint Policy and Least-Privilege Access Control](1.8-Week8/)

**Week 9:** [Monitoring, Logging, and Troubleshooting Connectivity](1.9-Week9/)

**Week 10:** [Cost Optimization and Resource Cleanup Automation](1.10-Week10/)

**Week 11:** [Documentation, Blog Translation, and Knowledge Sharing](1.11-Week11/)

**Week 12:** [Final Demo, Reporting, and Self-Evaluation](1.12-Week12/)
